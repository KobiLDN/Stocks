# AI Infrastructure Stock Dashboard

A personal investment research dashboard tracking 48 AI infrastructure stocks in GBP, with live price updates, fundamental metrics, TradingView charts, and a what-if return calculator.

**Live site:** https://KobiLDN.github.io/aiSTOCKS/

## Pages

### üìä Dashboard (`index.html`)
- 48 stocks across 12 categories
- Prices in GBP with 52-week range bars, 1D/1W/1M change pills, YTD and 1yr return
- Sortable columns: Ticker, Company, Category, Price, YTD, 1yr Return
- Filter buttons per category; ticker/company search box
- Ticker symbols link to Yahoo Finance

### üìã Metrics (`metrics.html`)
- Fundamental metrics per stock: market cap (¬£B), beta, P/E, average volume, 1D/1W/1M volume, YTD, dividend yield, short interest, analyst rating
- Sortable columns; category filter; search; hover tooltips on every column header

### üìà Charts (`charts.html`)
- TradingView embedded charts for every stock
- **Single stock** view with sidebar list
- **Category grid** view ‚Äî all stocks in a category at once
- Periods: 1W / 1M / 1Y / 5Y
- Mobile-friendly with collapsible sidebar

### üì∞ News (`news.html`)
- Browsable feed of the latest headlines across all 48 stocks
- Sentiment scored locally with VADER (green = positive, red = negative, grey = neutral) ‚Äî keyless, no API
- Sentiment-coloured cards: ticker (Yahoo link), company, category badge, time-ago, publisher
- Sort by newest / oldest / most positive / most negative; category filter; search
- Renders from `prices-data.js` at runtime (see AGENTS.md re: data freshness)

### üéØ Signals (`signals.html`)
- **AI top 10 ranked picks** with rationale and driver chips
- Generated by a **local** LLM (`qwen3.5-9b-claude-4.6-opus-reasoning-distilled-v2` running in LM Studio) ‚Äî keyless, no cloud API
- Each card: rank, ticker (Yahoo link), company, category, signal pill (STRONG BUY / BUY), confidence bar, rationale, driver chips
- Refreshed manually by running `python generate_signals_local.py` against LM Studio on `localhost:1234`
- Output committed as `signals-local.json` + `signals-local.js` so the live page has data immediately on push
- **Not refreshed by CI** ‚Äî see AGENTS.md "Local-only scripts" for why

### üßÆ What-If (`calculator.html`)
- Jan 2026 investment simulation ‚Äî current value of ¬£100 / ¬£1k / ¬£10k per stock based on YTD return
- Sortable columns; totals row; search box

## Price Updater Bot

Prices are kept live automatically via a GitHub Actions workflow (`.github/workflows/update-prices.yml`).

**Schedule:** twice daily on weekdays
- `13:00 UTC` ‚Äî pre-US-market snapshot (~09:00 ET / 14:00 BST)
- `21:30 UTC` ‚Äî 30 min after US close (~17:30 ET / 22:30 BST)

**What it does:**
1. Fetches live prices for all 48 tickers from Yahoo Finance via `yfinance`
2. Fetches live GBP/USD and GBP/JPY FX rates
3. Converts all prices to GBP
4. Fetches fundamental metrics (market cap, beta, P/E, volumes, dividend yield, short interest, analyst rating)
5. Fetches recent news headlines per ticker and scores each locally with VADER (keyless ‚Äî no API)
6. Writes prices, 52-week ranges, returns, change pills, metrics, and news into `index.html`, `prices.json`, and `prices-data.js`
7. Commits and pushes all three data files to `main` if anything changed

> ‚öÝÔ∏è Pages like `metrics.html` and `news.html` render from `prices-data.js` at runtime. New bot output fields won't appear on the live site until the workflow regenerates that file on GitHub ‚Äî see `AGENTS.md`.

**FX handling:**
- USD stocks ‚Üí divided by GBP/USD rate
- LSE stocks (RR.) ‚Üí quoted in pence, divided by 100
- Tokyo stocks (285A) ‚Üí divided by GBP/JPY rate

**To run manually:** trigger via GitHub Actions ‚Üí `Update Prices` ‚Üí `Run workflow`, or run `python update_prices.py` locally.

## Local development

The repo is cloned twice into a container folder ‚Äî two **independent clones**, no git worktrees:

```
aiStocks/
‚îú‚îÄ‚îÄ aiStocksMAIN/   ‚Üê clone on `main` ‚Äî the safe copy + what deploys live
‚îî‚îÄ‚îÄ aiStocksDEV/    ‚Üê clone on `dev`  ‚Äî the workbench, where edits happen
```

**3-step workflow:**

1. **Edit in `aiStocksDEV`** ‚Üí commit ‚Üí `git push origin dev`
2. **Sync with `aiStocksMAIN`** ‚Üí `git fetch origin && git merge --ff-only origin/dev`
3. **Save to GitHub** ‚Üí `git push` from `aiStocksMAIN` (this updates the live site)

`main` is always a known-good restore point ‚Äî if `dev` is trashed, `git reset --hard origin/main` or just re-clone. See `AGENTS.md` for full detail.
